# 🚀 CyberGate RVG - سیستم مدیریت گیت‌وی پیشرفته

**CyberGate RVG** یک پروژه جامع برای مدیریت تونل‌های VLESS over WebSocket (WSS) با یک داشبورد مدیریتی مدرن و تم سایبرپانک است. این سیستم برای دیپلوی سریع روی **Railway** بهینه شده است.

## ✨ ویژگی‌های کلیدی

- **پروتکل:** VLESS over WebSocket (WSS) روی پورت 443.
- **داشبورد:** رابط کاربری خیره‌کننده با تم سایبرپانک (Glassmorphism, Neon Effects).
- **مدیریت:** ساخت لینک‌های نامحدود، تعیین سقف ترافیک، فعال/غیرفعال سازی لحظه‌ای.
- **مانیتورینگ:** نمایش ترافیک مصرفی به صورت نمودار و مشاهده کاربران آنلاین در لحظه.
- **امنیت:** احراز هویت با JWT، هش رمز عبور با bcrypt، کنترل دسترسی ادمین.
- **دیتابیس:** ذخیره‌سازی دائمی در PostgreSQL و کشینگ سریع با Redis.

## 🛠 تکنولوژی‌های مورد استفاده

- **Backend:** FastAPI (Python 3.11), SQLAlchemy, PostgreSQL, Redis.
- **Frontend:** React 18, Vite, Tailwind CSS, Framer Motion, Recharts.
- **Deployment:** Docker (Multi-stage Build), Railway.app.

## 🚀 راهنمای دیپلوی روی Railway

1. **GitHub:** ابتدا این پروژه را در یک ریپازیتوری خصوصی یا عمومی در گیت‌هاب آپلود کنید.
2. **Railway Project:** در پنل Railway، یک پروژه جدید ایجاد کرده و آن را به ریپازیتوری گیت‌هاب متصل کنید.
3. **Databases:** دو سرویس زیر را به پروژه اضافه کنید:
   - **PostgreSQL**
   - **Redis**
4. **Environment Variables:** متغیرهای زیر را در بخش Variables تعریف کنید:
   - `DATABASE_URL`: (از دیتابیس Postgres ریلی را کپی کنید)
   - `REDIS_URL`: (از سرویس Redis ریلی را کپی کنید)
   - `JWT_SECRET`: یک رشته تصادفی طولانی (مثلاً خروجی `openssl rand -hex 32`)
   - `ADMIN_EMAIL`: ایمیل ادمین (مثلاً `admin@cybergate.local`)
   - `ADMIN_PASSWORD`: رمز عبور قوی برای ادمین
   - `GATEWAY_DOMAIN`: دامنه اختصاصی ریلی شما (مثلاً `cybergate.up.railway.app`)
5. **Deploy:** ریلی به صورت خودکار Dockerfile را شناسایی کرده و پروژه را بیلد و اجرا می‌کند.
6. **Initialize:** پس از اولین دیپلوی، برای ساخت دیتابیس و کاربر ادمین، دستور زیر را در محیط Shell ریلی اجرا کنید:
   `python scripts/init-db.py`

## 📂 ساختار پروژه

- `/backend`: منطق سرور، APIها و مدیریت ترافیک.
- `/frontend`: رابط کاربری با استایل سایبرپانک.
- `/scripts`: اسکریپت‌های کمکی برای مقداردهی اولیه.
- `Dockerfile`: پیکربندی داکر برای بیلد بهینه.

## 🛡 امنیت

- تمامی رمزها با الگوریتم **bcrypt** هش شده‌اند.
- توکن‌های دسترسی با **JWT** صادر می‌شوند و دارای تاریخ انقضا هستند.
- سیستم دارای مکانیزم **Healthcheck** برای اطمینان از پایداری سرویس است.
